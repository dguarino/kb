BrainScaleS Knowledge-Base

Main targets of this application are:

  • Improve communication between modelers and experimenters
  • Collect and share guideline statements and evidence from partners
  • Manage links among these data and selected bibliographical references
  • Collective evaluation of these data for their relevance inside the project

Having these points in mind, tha application has been centered around three basic concepts, which form the knowledge-base Map:

  • Statement: a sentence or (very) short paragraph, containing a bit of knowledge (to be) agreed among BrainScaleS Partners
  • Evidence: quantitative contributions (data, figures, measurements) supporting or contrasting one or more statements
  • Article: references to publication containing one or more evidence

The remaining of this page shows some detail of the features to create and manage knowledge items.

User Based approach

Access to the application is given under request, then the newly added user can access the website using username and password supplied. Afterwards a new password can be chosen.

To ease interaction with the system, few operation are allowed and (when permitted) always ready to the user, represented just by icons to keep the interface as clean as possible:

  • Creation button: it opens a dialog floating on top of the page to insert data.
  • Editing button: it opens the same dialog populated with already existing data.
  • Deleting button: it enables the user to hide the item (real deletion is left on the admin side, to ensure no data are lost).
  • Inspecting button: when available, it opens a new browser tab, containing all data for the item chosen. It is a non-interactive page, in order to avoid multiple path to access and produce the same information.

Questions: requests for Statements

In order to enlarge the knowlegde and shape statements, a 'request for statement' approach has been chosen, resembling stackoverflow.com, a community-driven, programming question&answer website.
Once logged-in, users land into a Questions page. Here all asked question are showed, together with eventual links to statements answering the question (and a counter for these answers).

Map: the current state

The page showing the actual Knowledge-Base has a three columns layout, one for each main object.i
Items for each type are initially listed by their publication date, but they can be selected (by clicking) leading to an highlighted, reorganized layout. The selected item is brought on top of its list, colored in yellow, and related items on the other lists are given the same appearace. This behaviour works for each column, enabling the user to see relations starting from each type of object.

Map details

Creation, editing, deletion and inspection of item on the map follow the same basic rules: if you created them, you can edit and destroy, otherwise you can just highlight, inspect and comment (see below).
In case the selected item is a statement, it gets highlighted together with all its related. A particularity of selecting statements is that supporting evidence are yellow, while contrasting evidence are red instead, thus giving a clear idea of the status in which the statement is.

Editing Relations

Statements have no relation to be specified on their side, being the final point of discussions and evaluation of evidence. Articles as well are standing alone, being just references. On the contrary, evidence do have relations, both towards articles they can have been taken from and towards statements, which they can support or contrast.

Topics

During statement creation (strongly suggested) but also in any other moment, it is possible to assign one or more topic tag to the statement. This ensures you and others to be able to find all relevant statement for a certain topic with just one click (see filters below).

Filters

On top of each list is present a field with two icons:      filter and      remove filter. They just set and remove words used to filter out those items in the corresponding list not having them. In case more than one word are supplied as filter, they are searched in OR mode.
Each time a filter is applied on a column, it propagates through relation to the other columns in AND mode, summing up their effects.
Also topics can be used to set filters for the list of statements, simply click on one of them to sets its words as filters, but be careful, since clicking on them replaces already existing filters for statement column!

Comments

Once logged in, a user can comment each item. All comments will be listed inside each item fold and only the total amount of comments is shown at the bottom left of the item card. Comments list include username and date for each comment as well as a simple box to add another comment to the same item. To read them better, they are shown also inside detail page of the item.
Comments can be removed only by administrators, so think before writing...
